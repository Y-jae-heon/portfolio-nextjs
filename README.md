# Portfolio Website

> 문제를 구조화하고 실행 가능한 시스템으로 연결하는 개발자의 포트폴리오

---

## Overview

이 프로젝트는 단순한 소개 페이지가 아니라,  
**“이 사람이 실제로 어떤 문제를 어떻게 해결하는지”를 보여주기 위한 포트폴리오 웹사이트**입니다.

- 단순 기술 나열 ❌
- 실제 문제 해결 방식 중심 구성 ✅
- 프로젝트 기반 설득 구조 ✅

---

## Purpose

이 포트폴리오는 아래를 명확하게 전달하기 위해 만들어졌습니다:

- 복잡한 요구를 구조화하고 실행으로 연결하는 능력
- 프론트엔드부터 아키텍처, AI 기능까지 End-to-End 경험
- 실제 운영 환경에서 문제를 해결한 사례

---

## Structure

웹사이트는 다음과 같은 구조로 구성되어 있습니다:

### 1. Hero

- 한 줄로 정의하는 개발자 정체성
- 실행력 중심 메시지

### 2. Summary

- 어떤 문제를 해결하는 사람인지
- 어떤 환경에서 강점을 가지는지

### 3. Featured Projects

- 대표 프로젝트 3개
- Problem / Role / Approach / Impact 구조

### 4. Capabilities

- 기술이 아닌 “팀에 기여하는 방식” 중심 정리

### 5. Timeline

- 단순 경력이 아닌 성장 흐름 중심 구성

### 6. Philosophy

- 실제 의사결정 기준과 문제 해결 방식
