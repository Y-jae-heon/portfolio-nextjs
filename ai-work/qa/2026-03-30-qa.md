## Issues Found

### Issue #1: ProjectCard impact prop — 빈 배열 시 undefined 전달 가능
- **Severity**: HIGH
- **Location**: `app/page.tsx` line 95, 301
- **Description**: `ProjectCard`의 `impact` prop이 `string`으로 선언되어 있으나 호출부에서 `project.impact[0]`을 전달. TypeScript는 `noUncheckedIndexedAccess` 없이 배열 인덱스 접근을 항상 `T`로 추론하므로 빈 배열일 때 `undefined`가 전달되어도 컴파일 타임에 잡히지 않음. 런타임에서 React가 `undefined`를 렌더링하면 성과 카드 내용이 공백으로 표시되거나 예상치 못한 출력이 발생할 수 있음.
- **Reproduction Steps**: `content.projects`에 `impact: []`인 프로젝트를 추가 → ProjectCard 성과 영역에 공백 또는 'undefined' 텍스트 렌더링
- **Expected vs Actual**: 빈 배열 → 성과 없음 처리(카드 숨김 or 공백 표시), 실제 → undefined가 string prop으로 전달
- **Recommendation**: 옵션 A — prop 타입을 `string | undefined`로 바꾸고 `{impact && <p>…</p>}` 조건 렌더링 적용. 옵션 B — 호출부에서 `impact={project.impact[0] ?? ""}` 폴백 처리. 현재 데이터 3개 모두 impact 배열 1개 이상이므로 즉각적 영향은 없으나 향후 콘텐츠 추가 시 조용히 깨질 수 있음.
