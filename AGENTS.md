# AGENTS.md

## Project Mission

Build a static-first portfolio website that presents the developer as a high-signal, senior-level product engineer with end-to-end capability across frontend, backend-adjacent architecture, infrastructure awareness, and AI-driven product design.

The site must feel professional, sharp, modern, and execution-oriented.
It should not read like a resume dump.
It should read like a compact product case-study website that increases interview conversion.

## Primary Goal

Maximize hiring confidence by showing:

1. problem-solving depth
2. architecture thinking
3. strong execution across product delivery
4. clear communication
5. polished frontend craftsmanship

## Constraints

- No backend is required.
- All content is static or local-content driven.
- Must be deployable on Vercel Hobby.
- Prefer Next.js with a static-first architecture.
- Avoid unnecessary complexity or server dependencies.
- Optimize for readability, performance, and strong first impression.
- Mobile and desktop responsiveness are mandatory.
- Accessibility basics are mandatory.
- Use clean typography, strong spacing, and restrained visual emphasis.
- Do not create a flashy toy portfolio.
- Do not use gimmicky animations that hurt trust.

## Portfolio Positioning

This portfolio should position the owner as:

- an end-to-end product-focused engineer
- a systems thinker
- a technical lead candidate
- a builder who can bridge AI, platform architecture, and frontend execution

## Required Sections

- Hero
- Professional summary
- Featured projects (3)
- Capability map
- Career timeline
- Problem-solving philosophy
- Contact / external links

## Writing Rules

- Use concise and assertive language.
- Prefer evidence over adjectives.
- Show impact with structure and outcomes.
- Avoid exaggerated self-praise.
- Every major project should communicate:
  - problem
  - role
  - approach
  - impact
  - stack

## Design Rules

- Minimal but high-end
- Strong hierarchy
- Clear scanning structure
- Distinct section rhythm
- Premium spacing
- High contrast, restrained palette
- Subtle motion only when it improves clarity
- Choose UI Framework ShadCn

## Engineering Rules

- Use reusable components
- Keep content typed and structured
- Prefer server-free static rendering
- Keep Lighthouse-friendly implementation
- Avoid overengineering
- Integrate a multi-agent orchestration by leveraging the predefined configurations in ./codex/agents and specialized capabilities in ./codex/skills to automate and modularize development tasks.

## Review Standard

A result is acceptable only if:

- the owner looks credible within 8 seconds
- the site feels like a serious engineer's portfolio
- the content is easier to understand than a PDF resume
- the featured projects read like strong case studies
- the site is ready for Vercel deployment without backend work
