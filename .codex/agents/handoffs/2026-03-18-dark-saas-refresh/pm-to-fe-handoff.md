# Handoff: PM to FE

## 구현 목표

정적 구조를 유지하면서 전체 포트폴리오를 dark SaaS 톤으로 리프레시한다.

## 필수 유지 사항

- 기존 섹션 순서
- `app/content.ts` 타입과 콘텐츠 구조
- 접근성 기본값
- Vercel 배포 가능성

## 우선순위

1. 글로벌 토큰과 배경
2. Hero와 핵심 성과 패널
3. Project 카드와 상세 블록
4. Timeline / Philosophy / Contact 정리
5. 모바일 밀도와 focus 상태

## 구현 기준

- 블루는 강조 포인트에만 사용
- 텍스트는 밝은 회색 단계로 위계화
- 배경은 어둡지만 피로하지 않게 처리
- 카드 반복 감각을 줄이기 위해 패널 강도를 구분
