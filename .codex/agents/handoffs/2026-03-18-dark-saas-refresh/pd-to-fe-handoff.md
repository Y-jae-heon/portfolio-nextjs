# Handoff: PD to FE

## 비주얼 시스템

- Page background: 딥 네이비 + 약한 블루 글로우
- Surface: navy slate 계열 2~3단계
- Border: 밝은 회색이 아닌 낮은 채도의 slate border
- Accent: 링크, CTA, 수치, active point에만 사용

## 타이포 적용

- Hero headline은 크고 단단하지만 과한 압축 금지
- section eyebrow는 mono + 작은 양의 tracking
- body copy는 line-height를 늘려 한국어 읽기 피로를 줄일 것
- detail label은 짧은 meta label 수준으로만 강조

## 컴포넌트 지침

- Button: rounded-full, primary/secondary 대비 분명
- Badge: pill shape, 작은 mono 메타 정보
- Project detail: 좌우 패널 분리로 스캔성 확보
- Timeline: 점과 라인의 대비를 억제하고 내용이 먼저 보이게

## 반응형 노트

- 모바일에서는 hero CTA와 chip이 2줄 이상 자연스럽게 떨어져야 함
- metric, badge, detail card가 과밀하지 않도록 padding 유지
