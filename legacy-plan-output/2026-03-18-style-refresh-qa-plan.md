# QA Plan: Style Refresh

## 목표

- 리브랜딩 후 사이트가 더 고급스러워졌는지보다, 더 신뢰 가능하고 더 읽기 쉬워졌는지를 검증한다.
- 시각 변경이 접근성, 반응형, 콘텐츠 우선순위, 배포 안정성을 해치지 않았는지 확인한다.

## 검증 범주

- Product clarity: 첫 화면에서 역할과 실적이 바로 보이는가
- Typography: Pretendard/JetBrains Mono 적용이 일관되고 과하지 않은가
- Color system: 검정/흰색/회색 기반이 안정적으로 유지되는가
- UX rhythm: 섹션 간 리듬과 카드 밀도가 개선되었는가
- Accessibility: 대비, focus, reduced motion, 모바일 탭 영역이 유지되는가
- Regression: 기존 주요 섹션과 CTA가 사라지거나 약해지지 않았는가

## 실패 조건

- Hero가 더 단조로워졌지만 더 명확해지지는 않은 경우
- Mono가 과해서 문서보다 개발자 밈처럼 보이는 경우
- 회색 대비가 낮아 설명 텍스트 가독성이 떨어지는 경우
- 빨강이 여러 위치에 퍼져 전체 화면에서 경고 UI처럼 보이는 경우
- 카드와 섹션 구분이 너무 약해져 정보 덩어리가 무너지는 경우

## 체크리스트

- 데스크톱과 모바일에서 헤더, Hero, 프로젝트 카드, 타임라인, 연락 섹션을 각각 확인한다.
- hover 없이도 모든 핵심 정보가 식별 가능한지 확인한다.
- 키보드 포커스가 회색 체계에서 충분히 보이는지 확인한다.
- 선택 상태, 링크 상태, CTA 우선순위가 일관적인지 확인한다.
- build/lint와 주요 스크린샷 비교가 가능하도록 결과를 남긴다.
