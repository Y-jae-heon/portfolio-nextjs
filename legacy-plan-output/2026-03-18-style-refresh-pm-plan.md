# PM Plan: Style Refresh

## 목표

- 포트폴리오의 첫인상을 "화려한 개인 사이트"가 아니라 "실력으로 승부하는 시니어 엔지니어의 제품 문서"로 재정렬한다.
- 컬러와 폰트 변경이 장식 추가가 아니라 콘텐츠 전달력 강화로 읽히도록 방향을 고정한다.
- Apple-like한 절제감은 유지하되, 채용 맥락에서는 더 직접적이고 더 읽기 쉬운 정보 구조를 우선한다.

## 리브랜딩 포지셔닝

- 톤: sharp / credible / restrained / execution-first
- 첫 인상: "UI를 잘 만드는 사람"보다 "문제를 구조화하고 결과로 증명하는 사람"
- 시각 신호: 검정과 흰색 중심, 회색으로 정보 계층을 나누고 빨강은 경고가 아니라 높은 우선순위 강조에만 제한적으로 사용

## 섹션별 UX 우선순위

- Hero: 헤드라인과 핵심 지표를 더 빠르게 읽히게 만들고, 시선이 장식보다 문장과 숫자에 먼저 닿게 한다.
- Summary: 설명 밀도는 유지하되 카드 장식을 줄여 문장 간 위계와 여백으로 읽힘을 확보한다.
- Featured Projects: 카드의 개성을 줄이고 케이스 스터디 진입점으로서의 구조와 비교 가능성을 강화한다.
- Capability Map: 범주 간 차이를 색이 아니라 구조와 타이포 위계로 구분한다.
- Timeline: 연대기 나열보다 역할 변화와 책임 확장을 드러내는 리듬으로 정리한다.
- Philosophy: 선언형 카피보다 의사결정 원칙이 보이게 만든다.
- Contact: CTA는 적게 두고, 신뢰 가능한 연결 수단을 바로 찾을 수 있게 한다.

## 수용 기준

- 첫 8초 안에 역할, 강점, 실행력, 대표 성과 수치가 모두 식별 가능해야 한다.
- 페이지 전반에서 검정/흰색/회색만으로도 정보 구조가 성립해야 하며, 빨강은 없어도 UX가 무너지지 않아야 한다.
- Pretendard가 본문 가독성을 책임지고, JetBrains Mono는 숫자, 라벨, 메트릭, 강조 키워드에만 의도적으로 사용되어야 한다.
- UI가 조용해진 만큼 프로젝트와 성과 문장이 더 강하게 읽혀야 한다.
- 현재 요구 섹션을 모두 유지하되, 시각적 리듬은 더 단단하고 더 절제된 방향으로 바뀌어야 한다.

## PD Handoff 요구사항

- 색상 토큰을 기본 4색 체계로 재구성하고, 각 토큰의 사용 금지 범위까지 명시한다.
- Pretendard/JetBrains Mono의 역할 분담, 크기 체계, 자간, 숫자 강조 규칙을 제안한다.
- 섹션별 시선 흐름, 카드 밀도, 경계선 사용량, 여백 리듬을 재설계한다.
- Apple-like라는 추상 표현을 실제 화면 규칙으로 번역한다.

## FE Handoff 요구사항

- `app/globals.css`와 `app/layout.tsx`에서 폰트/컬러 시스템 교체 경로를 분리해 정리한다.
- 기존 컴포넌트가 새 토큰 체계에 자연스럽게 매핑되는지 확인하고, 필요하면 variant를 줄여 단순화한다.
- Hero metric, badge, section eyebrow, CTA처럼 Mono가 들어갈 지점을 명시적으로 분류한다.
- 구현 이후 QA가 확인할 수 있도록 변경 범위를 토큰, 타이포, 레이아웃, 컴포넌트 네 범주로 나눈다.
