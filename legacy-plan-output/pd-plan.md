# PD Plan

## 목표

- 과장되지 않지만 고급스럽고, 실행력이 느껴지는 시각 체계를 정의한다.
- 한국어 콘텐츠가 많은 환경에서도 스캔성이 유지되는 레이아웃을 설계한다.
- 실제 사용자 노출 텍스트는 한국어 중심으로 적용하되, 한국인이 익숙하게 이해하는 영어 기술 용어와 직무명은 제한적으로 허용한다.
- UI 프레임워크는 `shadcn/ui` 기준으로 잡고, 정적 우선 구조 안에서 필요한 프리미티브만 사용한다.

## 수행 범위

- 색상, 타이포, 간격, 카드 체계 정의
- 섹션별 배치 원칙 정의
- 모바일/데스크톱 레이아웃 규칙 정의
- 모션 및 접근성 기준 정의
- `Button`, `Card`, `Badge`, `Separator`, `Accordion` 계열 컴포넌트 활용 기준 정의

## 산출물

- 비주얼 시스템
- 레이아웃 로직
- UI 품질 기준
- 금지 패턴 목록

## 검수 기준

- 템플릿 포트폴리오처럼 보이지 않아야 한다.
- 한국어 헤드라인과 본문이 답답하지 않게 읽혀야 한다.
- 히어로, 프로젝트, 역량, 타임라인, 철학, 연락 섹션의 리듬이 분명해야 한다.
- 장식보다 정보 밀도가 우선되어야 한다.
