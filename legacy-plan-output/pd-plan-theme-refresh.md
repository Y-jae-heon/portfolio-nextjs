# PD Plan: Theme Refresh

## 목표

- 화려함을 줄이고 신뢰를 올리는 시각 시스템으로 재설계한다.
- 한국어 장문 콘텐츠가 많은 구조에서도 스캔성이 유지되도록 타이포와 여백 리듬을 재조정한다.
- "실력으로 승부"라는 인상이 UI 장식이 아니라 정보 구조에서 나오게 만든다.

## 디자인 컨셉

- Quiet Precision
- 흑백 대비, 얇은 회색 구분, 제한된 붉은 포인트로 긴장감을 만든다.
- 카드와 배경 효과를 줄이고, 활자 밀도와 정렬 정확도로 완성도를 만든다.

## 색상 규칙

- `#000000`: 헤드라인, 핵심 CTA, 주요 수치, 주요 경계의 기준색
- `#FFFFFF`: 페이지 배경, 카드 기본면, 여백의 호흡 유지
- `#6B7280`: 본문 보조, eyebrow, 설명 텍스트, 보더, 분리선
- `#EF4444`: 상태성 강조, 핵심 임팩트 숫자, 선택적 링크 포인트

## 타이포 규칙

- H1/H2/H3, 본문, 버튼, 내비게이션: `Pretendard`
- 수치, 프로젝트 메타, stack, eyebrow, 라벨, inline 강조: `JetBrains Mono`
- 제목은 두껍게 가되 과한 자간 압축을 줄이고, 본문은 행간으로 읽기 품질을 확보한다.

## 레이아웃 원칙

- Hero는 좌우 분할은 유지하되, 우측 카드 장식을 줄이고 숫자/문장 대비로 정리한다.
- 각 섹션의 도입부 폭을 조금 더 좁혀 제목 집중도를 높인다.
- 프로젝트 카드는 "배경 박스"보다 "케이스 인덱스"처럼 보이게 밀도와 정렬을 재설계한다.
- 회색 보더와 넉넉한 상하 패딩으로 섹션 리듬을 만든다.

## 컴포넌트 조정

- `Badge`: 더 얇고 정제된 라벨형으로 변경, 배경색 의존도 축소
- `Card`: 그림자보다 얇은 보더와 미세한 톤 차이로 구분
- `ButtonLink`: 검정 실면 버튼 + 흰 바탕 아웃라인 버튼의 2종 체계로 단순화
- `Separator`: 구조 구분 역할을 더 분명히 하고, 장식 요소 역할은 제거
- Mobile Nav: 둥근 "버블" 느낌을 줄이고 더 평평하고 정돈된 메뉴로 교체

## 모션 및 접근성 기준

- reveal 모션은 초기 진입부와 CTA 정도에만 제한
- hover 이동량과 그림자 증가는 최소화
- 흑백 대비에서 WCAG 대비가 떨어지지 않도록 회색 텍스트 밝기를 엄격히 관리
- focus ring은 검정/회색 체계와 충돌하지 않는 선명한 outline으로 유지

## 금지 패턴

- 푸른 계열 포인트 재사용
- 과한 배경 그라데이션
- 카드 내부 중첩 카드 남발
- Mono 폰트를 본문 대량 사용
- 붉은 색을 링크/강조 전반에 상시 노출
