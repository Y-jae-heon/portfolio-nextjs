# QA Plan

## 확인 항목

- 텍스트 대비가 충분한가
- mono 사용 범위가 과하지 않은가
- Hero / Projects / Timeline의 위계가 명확한가
- 모바일에서 카드와 태그가 과밀하지 않은가
- sticky header와 focus ring이 다크 배경에서 충분히 보이는가

## 실패 조건

- 첫 화면이 여전히 템플릿처럼 보임
- 대표 프로젝트가 강한 사례처럼 읽히지 않음
- 한국어 본문이 답답하거나 자간이 부자연스러움
- hover/focus/CTA 구분이 약함

## 결과물

- 데스크톱/모바일 캡처
- 이슈 목록
- 수정 권고 또는 배포 가능 판정
