# PD Plan: Style Refresh

## 목표

- 검정과 흰색 중심의 고대비 체계로 신뢰감을 높이고, 장식성보다 정보 전달력을 강화한다.
- Pretendard와 JetBrains Mono 조합을 통해 한국어 가독성과 개발자 정체성을 동시에 확보한다.
- 기존 블루 기반 장식과 패턴 배경을 줄이고, 시각적 자신감은 간격과 정렬, 계층으로 만든다.

## 컬러 시스템

- Primary: `#000000`
- Secondary: `#FFFFFF`
- Accent: `#6B7280`
- Optional Accent: `#EF4444`

## 컬러 사용 원칙

- 배경은 거의 흰색 단일 계열로 유지하고, 구획 분리는 면보다 border와 spacing으로 해결한다.
- 본문과 제목은 검정을 기본으로 하고, 설명 텍스트와 보조 정보만 회색으로 내린다.
- 빨강은 CTA 주색이 아니라 핵심 지표, 주의가 필요한 비교, 현재 상태 강조 같은 좁은 맥락에만 쓴다.
- 색 대비 없이도 읽히는 레이아웃이어야 하며, 색은 구조를 보조하는 수준에 머물러야 한다.

## 타이포 시스템

- 제목 / 본문 / 내비게이션 / 버튼: Pretendard
- 숫자 / 코드성 라벨 / eyebrow / 메트릭 / 짧은 강조어: JetBrains Mono
- Hero H1은 무게감 있는 Pretendard로 두고, Mono는 주변 정보와 수치 대비를 만드는 역할로 제한한다.
- 현재 `font-display` 사용 영역은 대부분 Pretendard로 흡수하고, display용 별도 감성 폰트는 제거한다.

## 레이아웃 재조정

- Hero: 왼쪽 카피와 오른쪽 성과 카드의 위계를 더 분명히 하고, 카드의 배경 차이를 줄여 숫자 자체가 먼저 보이게 만든다.
- Header: 배경 효과를 최소화하고, 얇은 border와 타이트한 spacing으로 정제한다.
- Section Intro: eyebrow, title, description 간 간격을 좁혀 스캔 효율을 높인다.
- Project Cards: hover 존재감은 낮추고, 구조적 일관성과 비교 가능성을 우선한다.
- Capability / Philosophy: 카드 덩어리보다 텍스트 블록 중심으로 재구성해 내용 밀도를 살린다.
- Timeline: 장식선보다 정렬과 들여쓰기 규칙으로 흐름을 만든다.

## 컴포넌트 가이드

- `Badge`: 회색 기반 저대비 pill로 축소하고 정보 태그처럼 동작시킨다.
- `ButtonLink`: 기본 버튼은 검정 바탕 흰 글자, 보조 버튼은 흰 바탕 검정 border로 단순화한다.
- `Card`: 큰 그림자 대신 얕은 border와 미세한 톤 차이만 사용한다.
- `Separator`: 섹션 구획 감을 만드는 핵심 요소로 활용하되 과용하지 않는다.

## 모션 및 인터랙션

- 기존 rise/fade는 유지 가능하지만 거리와 지속 시간을 줄여 존재감을 낮춘다.
- hover는 "떠오름"보다 "명확해짐" 쪽으로 조정한다.
- sticky header blur, 배경 패턴, 강한 highlight는 줄여 집중 분산을 막는다.

## 검수 기준

- 화면이 비어 보이지 않으면서도 시끄럽지 않아야 한다.
- Mono 사용량이 과해 보이지 않고, 정말 중요한 라벨과 숫자만 개발자적 인상을 줘야 한다.
- 컬러 존재감보다 문장과 구조가 먼저 기억나야 한다.
- 모바일에서도 좁은 화면 밀도가 유지되고 답답하지 않아야 한다.
