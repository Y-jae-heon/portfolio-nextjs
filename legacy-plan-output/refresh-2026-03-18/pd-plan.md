# PD Plan

## 디자인 컨셉

- "검은 잉크로 정리된 엔지니어링 문서"에 가까운 인터페이스를 만든다.
- 애플 스타일의 절제감은 가져오되, 제품 소개 사이트보다 엔지니어 포트폴리오의 정보 밀도와 명료성을 우선한다.

## 색상 규칙

- `#000000`: 주요 헤드라인, 핵심 CTA, 경계 강조, 중요한 수치
- `#FFFFFF`: 기본 배경과 카드 내부의 여백 유지
- `#6B7280`: 본문 보조 텍스트, 구획 보조선, 라벨, 메타 정보
- `#EF4444`: 경고가 아니라 "강한 주목"이 필요한 단일 포인트에만 사용

## 타이포그래피 규칙

- 제목과 본문은 모두 `Pretendard`로 통일한다.
- `JetBrains Mono`는 숫자, 성과 지표, eyebrow, stack, 코드성 메타에만 쓴다.
- Display 전용 개성 폰트는 제거한다.
- 자간과 크기 차이로 계층을 만들고, 색상으로 계층을 과하게 만들지 않는다.

## 레이아웃 리듬

- 섹션 간 간격은 넉넉하게 유지하되, 카드 내부는 더 단단하게 묶는다.
- Hero와 Projects는 수직 리듬보다 좌우 정렬의 안정감을 우선한다.
- 한 화면에 너무 많은 시각 장치가 겹치지 않도록 배경 표현을 줄인다.
- 장식 대신 구획선, 여백, 타이포 대비로 프리미엄 인상을 만든다.

## 컴포넌트 조정

- Badge: 칩 느낌보다 정보 태그 느낌으로 바꾼다.
- Card: 큰 그림자와 과한 라운드를 줄이고, 평면적인 밀도감을 높인다.
- Button: 채움 버튼과 아웃라인 버튼의 대비를 더 명확히 한다.
- Separator: 섹션 리듬을 만드는 주요 장치로 활용한다.
- Metric block: Mono 수치와 Pretendard 설명의 대비를 강화한다.

## 모션 및 접근성 가드레일

- 등장 애니메이션은 유지하더라도 거리와 속도를 줄인다.
- hover 효과는 이동보다 대비 변화 위주로 정리한다.
- 색상에만 의존한 정보 전달을 피한다.
- 대비는 기본적으로 WCAG AA 수준을 만족해야 한다.

## UX 리스크

- 애플 스타일을 과하게 해석하면 지나치게 비어 보일 수 있다.
- Mono 사용 범위가 넓어지면 읽기 피로가 커진다.
- 강조색 빨강을 여러 곳에 쓰면 전체 톤이 불안정해진다.
