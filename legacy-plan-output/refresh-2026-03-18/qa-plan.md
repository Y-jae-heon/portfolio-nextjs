# QA Plan

## 목표

- 스타일 리프레시가 신뢰감, 가독성, 반응형 완성도를 실제로 높였는지 검증한다.
- 단순 색상 변경이 아니라 정보 위계 개선으로 이어졌는지 본다.

## 검수 범위

- Hero 첫인상
- Projects 스캔성
- 폰트 적용 일관성
- CTA 강조 수준
- 모바일/데스크톱 레이아웃 안정성
- focus, hover, contrast 접근성

## 핵심 체크포인트

- Pretendard가 제목과 본문 전반에 일관되게 적용되는지
- JetBrains Mono가 숫자/메타/강조에만 제한적으로 쓰이는지
- 빨강 강조가 1차 포인트를 흐리지 않는지
- 카드, 배지, 버튼이 이전보다 더 절제되고 콘텐츠 중심인지
- 섹션 간 리듬이 자연스럽고 비어 보이지 않는지

## 실패 조건

- 블루 계열이나 이전 디스플레이 폰트 흔적이 남아 있다.
- 콘텐츠보다 UI 장식이 먼저 눈에 들어온다.
- Hero에서 핵심 메시지보다 스타일 요소가 튄다.
- 모바일에서 메타 텍스트나 수치 블록이 과밀하다.
- focus 상태가 약해 접근성이 저하된다.

## 산출물

- 시각 QA 메모
- 주요 뷰포트 스크린샷
- 수정 필요 항목 목록
