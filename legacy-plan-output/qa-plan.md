# QA Plan

## 목표

- 제품 메시지, 디자인, 프런트엔드 품질, 반응형, 배포 준비 상태를 검수한다.
- Playwright 기반 E2E 테스트와 실제 스크린샷을 남긴다.
- 실제 사용자 노출 텍스트가 한국어 중심으로 유지되는지 확인한다. 영어가 남아 있어도 한국인이 즉시 이해 가능한 수준인지 함께 검토한다.

## 수행 범위

- 개발 서버 실행
- Playwright 설치
- 핵심 섹션 렌더링 확인
- 반응형 뷰포트 점검
- 스크린샷 생성
- QA 결과 문서화
- 과도한 영문 UI 문자열 잔존 여부 확인

## 산출물

- E2E 테스트 코드
- 캡처 스크린샷
- `qa-report/` 검수 리포트
- `output-report/qa.md` 요약 리포트

## 검수 기준

- 페이지가 정상 렌더링되어야 한다.
- 주요 섹션과 CTA가 노출되어야 한다.
- 모바일과 데스크톱에서 레이아웃 붕괴가 없어야 한다.
- 빌드/배포에 치명적 문제가 없어야 한다.
- 과도한 영문 UI 문자열이 남아 있지 않아야 하며, 남아 있는 영어는 한국 채용 문맥에서 즉시 이해 가능해야 한다.
